# SOC Analyst Playbook

Welcome to the **Hydra SOC Analyst Playbook**. This guide details how to leverage the Hydra dashboard to supercharge Tier-1 alert triage, investigate anomalies, and securely authorize containment scripts.

## The React Dashboard Overview

Once authenticated, you will be presented with the main HUD. The navigation bar on the left grants access to:
- **Dashboard**: High-level SOC metrics, active investigations, and system load.
- **Investigations**: The historical backlog of all AI-driven triage runs.
- **New Investigation**: The entry point for manual or file-driven triage tasks.
- **Threat Intel**: The UI over the **Hydra Intelligence Fabric**.
- **Playbooks**: Curated workflows mapping common incident types to sequences of investigations.

## Initiating an Investigation

Investigations can be triggered automatically via Webhook (from your SIEM) or manually.

### Manual / File Upload
1. Navigate to **New Investigation**.
2. Select the Investigation Type (e.g., Log Analysis, Threat Hunt, Code Audit).
3. Provide a clear natural language prompt describing the anomaly.
4. *(Optional)* Upload a log file, PCAP, or structured CSV artifact up to 10MB.
5. Click **Launch Investigation**.

## Investigation Timeline and Risk Scores

Upon entering a specific task view, you will see the **Investigation Timeline**. 

Hydra does not operate chaotically. Using the **Hydra Intelligence Fabric**, it breaks down your prompt into discrete chronological steps:
- **Planning**: Retrieving relevant intelligence.
- **Synthesizing**: Analyzing logs against the Intelligence Fabric.
- **Code Generation**: Writing forensic scripts.
- **Execution**: Running the scripts in the Zero-Egress sandbox.

Each task culminates in a **Risk Score** generated by the AI, analyzing the artifacts derived from the sandbox execution.

## The Approval Gate (Human-in-the-Loop)

Hydra is completely deterministic, but for specific high-risk execution classes or unknown anomalies, it requires Human-in-the-Loop authorization.

When an investigation pauses with a `Pending Approval` status:
1. Review the "Generated Code" tab. The Python AST pre-filter ensures the code cannot escape, but you should review the logic to ensure it serves the forensic purpose.
2. Review the "Action Summary" highlighting exactly what the script intends to do with the supplied parameters.
3. Click "Approve" to authorize the cryptographic token, which releases the lock on the Temporal State Machine, allowing the Worker to proceed with execution.

## Leveraging Threat Intel (Intelligence Fabric)

The **Threat Intel** page showcases the current matrix of skills loaded into your tenant. Each entry denotes a specific Threat Vector (e.g., Lateral Movement, DDoS) mapped to MITRE ATT&CK tactics.

Skills labeled with an **Executable** badge mean Hydra possesses a strict Code Template for this methodology, completely bypassing generative code errors. Skills labeled **Methodology Only** rely on Hydra dynamically generating logic based on the textual methodology. 

Over time, utilizing these skills populates the **Episodic Security Memory**, refining the risk scores of future investigations based on local relevance.
