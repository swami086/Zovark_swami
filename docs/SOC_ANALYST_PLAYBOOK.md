# SOC Analyst Playbook

**Version: v1.5.1 | Date: 2026-03-24**

Welcome to the **Zovarc SOC Analyst Playbook**. This guide details how to leverage the Zovarc dashboard to supercharge Tier-1 alert triage, investigate anomalies, and securely authorize containment scripts.

---

## Dashboard Navigation

Once authenticated, the sidebar gives you access to the following pages:

| Page | What It Does |
|------|-------------|
| **Dashboard** | High-level SOC metrics, active investigation counts, and system health at a glance. |
| **Investigations** | Full task list with status, severity, and type filtering. Click any row to drill into the timeline. |
| **New Investigation** | Launch a manual investigation, upload a log/PCAP/CSV artifact (up to 10 MB), or select a predefined playbook. |
| **SIEM Alerts** | Real-time feed of alerts from connected log sources. Filter by severity or status, search by IP or rule name, and kick off investigations with one click. |
| **Entity Graph** | Visual map of entities (IPs, domains, users, hashes) and their relationships across investigations. |
| **Playbooks** | Browse, edit, and build multi-step response playbooks that chain investigation actions together. |
| **Skills** | Threat-intel skills library mapped to MITRE ATT&CK tactics. Skills marked **Executable** use strict templates; **Methodology Only** skills rely on generated logic. |
| **Admin Panel** | Tenant management, user management, pending approvals queue, and cost tracking dashboard (admin role required). |
| **Log Sources** | Configure SIEM webhook integrations, copy webhook URLs, and toggle auto-investigate per source. |

## Initiating an Investigation

Investigations can be triggered three ways:

1. **Manually** -- from the New Investigation page.
2. **From a SIEM alert** -- click "Investigate" on any alert row.
3. **Automatically** -- when a log source has auto-investigate enabled, incoming alerts launch investigations without manual intervention.

### Manual / File Upload

1. Navigate to **New Investigation**.
2. Select the Investigation Type (e.g., Log Analysis, Threat Hunt, Incident Response, Code Audit, IOC Scan).
3. Provide a clear natural-language prompt describing the anomaly.
4. *(Optional)* Upload a log file, PCAP, or structured CSV artifact up to 10 MB.
5. *(Optional)* Choose an existing playbook to guide the investigation steps.
6. Click **Launch Investigation**.

## Investigation Timeline and Risk Scores

Upon entering a specific task view, you will see the **Investigation Timeline**.

Zovarc breaks down your prompt into discrete chronological steps using the **Intelligence Fabric**:

- **Planning** -- Retrieving relevant intelligence and prior investigation context.
- **Synthesizing** -- Analyzing logs against the Intelligence Fabric.
- **Code Generation** -- Writing forensic scripts.
- **Execution** -- Running the scripts in the zero-egress sandbox (network disabled, syscalls restricted, 30-second kill timer).

Each task culminates in a **Risk Score** generated by the AI, analyzing the artifacts derived from the sandbox execution.

## The Approval Gate (Human-in-the-Loop)

For high-risk execution classes or unknown anomalies, Zovarc requires Human-in-the-Loop authorization before proceeding.

When an investigation pauses with a **Pending Approval** status:

1. Review the **Generated Code** tab. The Python AST pre-filter ensures the code cannot escape the sandbox, but you should review the logic to confirm it serves the forensic purpose.
2. Review the **Action Summary** highlighting exactly what the script intends to do with the supplied parameters.
3. Click **Approve** to authorize the cryptographic token, which releases the lock on the Temporal workflow and allows the worker to proceed with execution.

Pending approvals are also visible in the **Admin Panel** approvals queue, so team leads can review and approve on behalf of their analysts.

## Shadow Mode

Shadow Mode is Zovarc's safety-first automation protocol. Instead of taking actions directly, Zovarc generates a **recommendation** for every alert and waits for an analyst to make the actual decision.

### How It Works

1. An investigation completes and Zovarc produces a recommendation (e.g., *escalate*, *contain*, *dismiss*, *monitor*, *investigate further*) along with a severity assessment and confidence score.
2. The recommendation is shown to the analyst. Zovarc does not act -- **you decide**.
3. Your decision is recorded alongside the recommendation so the system can track how closely its suggestions align with real analyst judgment.

### Conformance Tracking

The conformance dashboard shows the **match rate** between Zovarc's recommendations and your team's actual decisions, broken down by action type. This gives your SOC leadership a clear, data-driven picture of how much the AI can be trusted before granting it more autonomy.

### Graduation Protocol

Zovarc can graduate through three automation levels, but only when the data supports it:

| Level | Requirements | Behavior |
|-------|-------------|----------|
| **Shadow** (default) | Always starts here | Zovarc recommends, human decides. |
| **Assisted** | 14+ days in shadow, 90%+ match rate, 50+ recorded decisions | Zovarc pre-fills the recommended action; analyst confirms or overrides. |
| **Autonomous** | 90+ days, 95%+ match rate, zero false negatives, 50+ decisions | Zovarc executes low-risk actions automatically; high-risk still requires approval. |

A **kill switch** is available at every level. If activated, all automation halts immediately and Zovarc reverts to shadow-only mode.

## SIEM Integration

Zovarc connects to your existing SIEM through webhook-based log sources. Supported platforms include Splunk, Elastic, QRadar, Syslog, and generic webhooks.

### Connecting a Log Source

1. Navigate to **Log Sources** (admin role required).
2. Click **Connect New Source**.
3. Fill in the source name, select the platform type, and optionally set an HMAC-SHA256 webhook secret for payload verification.
4. Toggle **Auto-investigate incoming alerts** if you want Zovarc to begin triage automatically when an alert arrives.
5. Select the default task type for auto-investigations (Log Analysis, Threat Hunt, Incident Response, Code Audit, or IOC Scan).
6. Click **Create Source** and copy the generated webhook URL into your SIEM's alert action configuration.

### Working with SIEM Alerts

Once a source is connected and sending alerts, the **SIEM Alerts** page provides:

- **Severity breakdown** -- top-line cards showing counts by Critical, High, Medium, and Low.
- **Filtering** -- narrow the feed by severity, status (New / Investigating / Resolved / Dismissed), or free-text search across alert names, IPs, and rule names.
- **One-click investigate** -- hit the green **Investigate** button on any alert row to launch a triage task immediately.
- **Bulk investigate** -- select multiple alerts and investigate them all at once.
- **Status tracking** -- alerts that already have a linked investigation show a **View** link that takes you straight to the task detail page.

### Alert Severity Filtering

Use severity filters to focus your attention where it matters most. A recommended workflow:

1. Start each shift by filtering to **Critical** and **High** alerts.
2. Investigate or bulk-investigate any that have not yet been triaged.
3. Work down through **Medium** alerts as time allows.
4. Review **Low** and **Informational** alerts periodically for pattern detection.

## Leveraging Threat Intel (Skills Library)

The **Skills** page showcases the current matrix of threat-intelligence skills loaded into your tenant. Each entry maps to a specific threat vector (e.g., Lateral Movement, DDoS) aligned to MITRE ATT&CK tactics.

- **Executable** skills have a strict code template, completely bypassing generative code errors.
- **Methodology Only** skills rely on Zovarc dynamically generating logic based on the textual methodology.

Over time, utilizing these skills populates the **Episodic Security Memory**, refining the risk scores of future investigations based on local relevance.
