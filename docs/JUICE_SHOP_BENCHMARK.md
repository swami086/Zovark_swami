## Analysis

### Strengths
- **100% attack detection** on all completed investigations
- **Consistent completion** for SQLi, XSS, path traversal, broken auth, IDOR, SSRF (50-63/63)
- **Fast triage** for most attack types (61-83s average)
- **Real traffic analysis** — the LLM generates code that correctly identifies attack patterns in actual HTTP request/response data

### Weaknesses
- **Benign traffic classification** — only 6/30 benign alerts completed (20%). Many timed out or failed. The benign alerts used generic task types that didn't match well with skill templates.
- **Command injection** — 0/5 completed. These alerts may not have matched any skill template.
- **JWT expiry** — 9/100 alerts failed with 401 errors. The benchmark runner's token refresh logic needs improvement.
- **False positive rate** — benign alerts that did complete had avg risk=53, which is above the detection threshold. ZOVARK currently errs on the side of flagging everything as suspicious.

### Completion Issues
- **Timeout (8)**: Investigations that took >5 min, typically because the LLM was generating complex code
- **Failed (14)**: Sandbox execution produced no output (template mismatch)
- **401 Errors (9)**: JWT token expired every ~5th request (15-min token lifetime × ~90s per investigation)

## Methodology

- **Target:** OWASP Juice Shop v17 (Docker, port 3001)
- **Corpus:** 100 alerts (70 attacks + 30 benign) generated by `attack_juice_shop.py`
- **Attack types:** SQL injection, XSS, path traversal, broken auth, IDOR, SSRF, file upload, command injection
- **Pipeline:** ZOVARK V2 (Ingest → Analyze → Execute → Assess → Store)
- **Model:** Qwen2.5-14B-Instruct Q4_K_M via llama.cpp (RTX 3050 4GB)
- **Detection threshold:** risk_score > 40 = threat detected
- **Spacing:** 15 seconds between submissions

## Conclusion

ZOVARK demonstrates **100% attack detection accuracy** on completed real-traffic investigations.
The primary challenge is completion rate (69%) — improving template coverage for uncommon
attack types and fixing JWT token management would bring this to 90%+.

For the attack types that ZOVARK handles well (SQLi, XSS, path traversal, auth attacks,
IDOR, SSRF), the system reliably detects threats in under 90 seconds on consumer hardware.
