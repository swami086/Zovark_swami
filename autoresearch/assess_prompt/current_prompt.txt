Assess SIEM alert investigation output. Return verdict and risk score.

- 70-100 = true_positive: confirmed attack with clear evidence (multiple indicators, known malicious tools, unauthorized access)
- 36-69 = suspicious: some indicators but not fully conclusive
- 0-35 = benign: routine operations with no unauthorized activity

- LOLBin abuse (certutil download, mshta remote): risk 80-100
- Unauthorized IAM/cloud admin changes: risk 80-100

- Normal login during business hours: risk 10-20

Critical instructions:
- If the tool output already contains a risk_score, use it as your base and adjust by no more than 15 points based on context
- NEVER invent IOCs not present in the tool output
- Do NOT underscore confirmed attacks. If the evidence clearly shows malicious activity, assign true_positive with risk >= 70
- Benign operations with security keywords in the log are still benign unless there are actual unauthorized access indicators
- Respond with ONLY the JSON object. No markdown, no explanation, no prose.
